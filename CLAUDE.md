## What This Is

UnionConfig is a set of F# libraries for type-safe configuration management using discriminated unions. Define your config vars as a DU, then read from environment variables, .env files, or AWS SSM Parameter Store with automatic parsing, validation, and secret masking.

Three packages:

- **UnionConfig** (core) -- types, env reader, .env file operations, verification
- **UnionConfig.Ssm** -- AWS SSM Parameter Store backend
- **UnionConfig.TextEditor** -- interactive text editor workflow for editing config (depends on CommandTree)

Test projects use xUnit v3 (standalone executables). `dotnet test` also works since `TestingPlatformDotnetTestSupport` is enabled.

## Architecture

### `src/UnionConfig/` (core)

- **Types.fs** -- Core types: `ConfigVarKind` (Manual, AutoGenerated, Infrastructure, AutoProvisioned), `ConfigValueType` (StringType, IntType, BoolType, FloatType, CustomType), `ConfigRequirement` (Required, Optional), `ConfigVarDef` record, `ConfigValue` DU, and the `ConfigValue` extraction module with typed helpers (`string`, `int`, `bool`, `float`, `custom` + option variants).
- **Verification.fs** -- `VerificationResult` DU (VerifySuccess, VerifyFailed, VerifySkipped) and colored terminal output for verification results.
- **Reader.fs** -- Reads config from environment variables. `read` returns `Result<ConfigValue option, string>`. Convenience functions: `readString`, `readInt`, `readBool`, `readIntOrDefault`, `readBoolOrDefault`. `validateRequired` checks all required vars are set.
- **EnvFile.fs** -- `.env` file operations: `readEnvFile`, `writeEnvFile` (sectioned output), `defaultSections` (builds sections from grouped defs), `compareConfigs` (diff detection), `maskValue` (secret masking), `openInEditor`, `displayChanges`.
- **ConfigRegistry.fs** -- Reflection-based DU case discovery: `allDefs` (flat), `allDefsGrouped` (grouped by wrapper case), `byName`, `names`.

### `src/UnionConfig.Ssm/`

- **SsmClient.fs** -- Low-level AWS SSM operations: `getParameter`, `getParametersByPath`, `setParameter`, `deleteParameter`. Configurable via `SsmClientConfig` (region, auth hook).
- **SsmConfigStore.fs** -- Higher-level store: `SsmConfigStore` record with path mapping and secret detection. Operations: `getValue`, `setValue`, `deleteValue`, `loadAll`, `applyChanges`.

### `src/UnionConfig.TextEditor/`

- **ConfigEditor.fs** -- Interactive editor workflow: `editConfig` (opens $EDITOR with .env temp file, diffs changes, verifies, confirms, applies) and `populateDefaults` (applies defaults without editor). Depends on CommandTree for UI helpers (spinners, colored output).

## Key Concepts

- **ConfigVarDef** -- The central type. Each config variable is defined as a record with name, kind, value type, requirement, secret flag, and documentation.
- **ConfigValueType** -- Determines parsing: StringType, IntType, BoolType, FloatType, or CustomType with a validator function.
- **ConfigRequirement** -- Required (fail on missing) vs Optional (return None).
- **ConfigVarKind** -- How a value is sourced: Manual (user-provided), AutoGenerated (with optional default), Infrastructure (set by infra), AutoProvisioned (set by scripts).
- **Reader** -- Reads from `Environment.GetEnvironmentVariable`, parses to typed `ConfigValue`. `read` returns `Result<ConfigValue option, string>`.
- **EnvFile** -- Read/write `.env` files, compare configs to detect drift, mask secrets for display. `writeEnvFile` + `defaultSections` generate sectioned `.env` files from grouped defs.
- **ConfigRegistry** -- Reflection-based discovery of all DU cases. `allDefs` returns flat array, `allDefsGrouped` returns `(groupName, defs)` pairs for nested DUs.
- **SsmConfigStore** -- Maps var names to SSM paths via `SsmPathMapping`, provides CRUD. `applyChanges` handles both sets and deletes.
- **ConfigEditor** -- Full edit workflow: load config -> write temp .env -> open editor -> diff -> verify -> confirm -> apply.

## Tests

- `tests/UnionConfig.Tests/` -- TypesTests.fs (parsing, ConfigValue extraction), ReaderTests.fs (env var reading, validation), EnvFileTests.fs (file parsing, diffing, masking)
- `tests/UnionConfig.Ssm.Tests/` -- SsmConfigStoreTests.fs (path mapping, delete detection logic; no live AWS calls)

Uses xUnit v3 + Unquote (`test <@ assertion @>` syntax). Core tests access internal members via `InternalsVisibleTo`.

## Code Style

- **Formatting**: Fantomas (auto-enforced)
- **Linting**: FSharpLint -- 4-space indent, 120 char max line, 1000 line max file, 500 line source length
- **Naming**: PascalCase for types/modules/records/unions, camelCase for values/params
- **Build strictness**: `TreatWarningsAsErrors` is enabled; `GenerateDocumentationFile` requires XML docs on public members
- **Target framework**: .NET 10
