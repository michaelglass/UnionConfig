## What This Is

UnionConfig is an F# library for type-safe configuration management using discriminated unions. Define your config vars as a DU, then read from environment variables, .env files, or AWS SSM Parameter Store with automatic parsing, validation, and secret masking.

One package (`UnionConfig`) with zero external dependencies.

## Architecture

### `src/UnionConfig/`

- **Types.fs** -- Core types: `ConfigVarKind` (Manual, AutoGenerated, Infrastructure, AutoProvisioned, External), `ConfigValueType` (StringType, IntType, BoolType, FloatType, CustomType), `ConfigRequirement` (Required, Optional), `ConfigVarDef` record, `ConfigValue` DU, and the `ConfigValue` extraction module with typed helpers (`string`, `int`, `bool`, `float`, `custom` + option variants), and the `ConfigVarKind` module with `isPersistable` for filtering env-edit / SSM-write flows.
- **Verification.fs** -- `VerificationResult` DU (VerifySuccess, VerifyFailed, VerifySkipped) and colored terminal output for verification results.
- **Reader.fs** -- Reads config from environment variables. `read` returns `Result<ConfigValue option, string>`. Convenience functions: `readString`, `readInt`, `readBool`, `readIntOrDefault`, `readBoolOrDefault`. `validateRequired` checks all required vars are set.
- **EnvFile.fs** -- `.env` file operations: `readEnvFile`, `writeEnvFile` (sectioned output), `defaultSections` (builds sections from grouped defs), `compareConfigs` (diff detection), `maskValue` (secret masking), `openInEditor`, `displayChanges`.
- **ConfigRegistry.fs** -- Reflection-based DU case discovery: `allDefs` (flat), `allDefsGrouped` (grouped by wrapper case), `byName`, `names`.
- **SsmConfigStore.fs** -- Generic config store backed by a parameter store. Consumers provide an `SsmOperations` record (GetParameter, SetParameter, DeleteParameter, GetParametersByPath). `SsmConfigStore` record combines operations with `SsmPathMapping` and secret detection. Functions: `getValue`, `setValue`, `deleteValue`, `loadAll`, `applyChanges`.
- **ConfigEditor.fs** -- Interactive editor workflow: `editConfig` (opens $EDITOR with .env temp file, diffs changes, verifies, confirms, applies) and `populateDefaults` (applies defaults without editor). Uses simple terminal output (no external dependencies).

## Key Concepts

- **ConfigVarDef** -- The central type. Each config variable is defined as a record with name, kind, value type, requirement, secret flag, and documentation.
- **ConfigValueType** -- Determines parsing: StringType, IntType, BoolType, FloatType, or CustomType with a validator function.
- **ConfigRequirement** -- Required (fail on missing) vs Optional (return None).
- **ConfigVarKind** -- How a value is sourced: Manual (user-provided), AutoGenerated (with optional default), Infrastructure (set by infra), AutoProvisioned (set by scripts), External (operator-set in env at runtime; never persisted, never editable). Use `ConfigVarKind.isPersistable` when iterating defs for env-edit / SSM writes.
- **Reader** -- Reads from `Environment.GetEnvironmentVariable`, parses to typed `ConfigValue`. `read` returns `Result<ConfigValue option, string>`.
- **EnvFile** -- Read/write `.env` files, compare configs to detect drift, mask secrets for display. `writeEnvFile` + `defaultSections` generate sectioned `.env` files from grouped defs.
- **ConfigRegistry** -- Reflection-based discovery of all DU cases. `allDefs` returns flat array, `allDefsGrouped` returns `(groupName, defs)` pairs for nested DUs.
- **SsmConfigStore** -- Maps var names to SSM paths via `SsmPathMapping`, provides CRUD. Consumers inject `SsmOperations` (no AWS SDK dependency). `applyChanges` handles both sets and deletes.
- **ConfigEditor** -- Full edit workflow: load config -> write temp .env -> open editor -> diff -> verify -> confirm -> apply.

## Code Style

- **Formatting**: Fantomas (auto-enforced)

- **Naming**: PascalCase for types/modules/records/unions, camelCase for values/params
- **Build strictness**: `TreatWarningsAsErrors` is enabled; `GenerateDocumentationFile` requires XML docs on public members
- **Target framework**: .NET 10
